Criterion Collection Online Help Center

The Criterion Collection’s help experience existed as long, unstructured FAQ pages that forced users to scroll endlessly for answers. My goal was to design a modern help centre that was scannable, mobile-friendly, category-based, and easy to navigate.

Tools:

Visual Studio Code Figma Miro

Industries:

Software Media/Streaming

My role

UX design,UX writing, Information architecture,User flow creation, Priority guides, Mobile-first content structure, Heuristics evaluation, and iteration across all prototypes.

View on GitHub Download MVP File

User Persona

Roger Avary, a busy cinephile who collects physical media and expects fast, dependable help when issues arise. He hates long FAQs, confusing labels, and slow support responses.

Process

1. Information Architecture & User Flow

We conducted three rounds of card sorting across diverse participants to refine category naming and grouping. Also, designed a userflow for consistent,predictable path.

Key Outcomes:

  • Removed vague labels like “Delivery Issue”.
  • Eliminated the ambiguous “Technical Support” category.
  • Consolidated Returns & Refunds with Order & Shipping.
  • Developed 5 final categories based on user expectations.

2. Priority Guides

Created content-first layouts for:

  • Help centre overview page
  • Category overview pages
  • Article pages

Emphasis placed on:

  • Plain language
  • Logical sequencing
  • Cross-linking
  • Minimalist hierarchy

3. Prototyping

Usability Testing: Version A

Average score: 7.5/10
Issues identified:

  • Search bar ignored
  • Contact Us dropdown confusing
  • Logo didn’t link home
  • Footer missing on some pages

Version B

Link to version B
Changes made:

  • Added “Contact Us” button
  • Removed search bar on article pages
  • Centered headings
  • Improved footer consistency
  • Simplified nav bar

Usability Testing: Version B

Average score: 8/10
Feedback:

  • Good readability
  • Want more images in articles
  • Logo should return home

4. Minimum Viable Prototype (MVP)

Key improvements:

  • Larger text spacing
  • Clickable logo returns home
  • "Back to Top" button separated
  • Tap/hover tile effects
  • Loading progress bar
  • Fully responsive mobile-first structure

Final Usability Rating:

9.3/10 (From same 8 users)

Impact

  • Users found the site cleaner, faster, and more intuitive.
  • FAQ frustration eliminated.
  • Navigation improved across all three iterations.
  • Mobile-first design significantly boosted scanability.
  • Content structure became consistent, predictable, and user-friendly.

Reflection

This project strengthened my skills in IA based on real user mental models.

  • Integrating UX writing with technical documentation structures.
  • Writing for mobile-first help experiences.
  • Structuring long-form content for clarity.
  • Iterative prototyping based on research.
  • Simplifying complexity into clear pathways.

If I were to redo this project, I would focus more on accessibility and inclusivity, refine the hover and tap interactions, and replace the current text-size buttons with a more flexible slider to support a wider range of users.
